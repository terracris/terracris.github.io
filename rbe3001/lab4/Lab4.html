Introduction

In our previous lab, we implemented Inverse Kinematics (IK) where we calculated the joint angles required to achieve a given end effector position. Additionally, we implemented trajectory generation techniques in the joint space and task space to create a smooth trajectory between two vertices using IK. In this lab, we will implement velocity kinematics, also known as differential kinematics. To do so, we will calculate the manipulator Jacobian matrix of the robot arm to calculate task-space velocities, identify the singular configurations, and implement an iterative numerical algorithm to solve the inverse kinematics problem iteratively. We will then compare trajectory generation using this approach to that of polynomial trajectories.

Development and Implementation

Deriving Jacobian for Velocity Kinematics

There are two ways to derive the Jacobian matrix for forward and inverse velocity kinematics: analytically and geometrically. We chose the analytic Jacobian solution due to its simplicity in implementation using Matlab. We took the partial derivatives of the end effector position from our transformation matrix with respect to each joint variable. This gave us a matrix of size 6x3. We devised the angular portion of the Jacobian J⍵ using the zi0 axes.

Figure 5: Jacobian and Jacobians determinant (Jv) for arm position [0,0,90]

In both Jacobians (Jv) generated in a singularity, the first columns are both all 0s. The first column of the Jv matrix corresponds to the changes to the transformation matrix based on theta 1. When the end effector is on the z-axis of joint 1, theta 1 and therefore any change in theta 1 does not have an impact on the end effector’s position.  The arm simply rotates about the axis without changing the end effector's x, y, or z positions.

The determinant is always 0 (or close to 0) because the first column of the Jv matrix is all 0s. As shown in the graphic below, if any of the columns are all 0 then the determinant will be equal to 0 since each of the 6 values summed together has a factor from each column.

Figure 6: Determinant calculation.

https://demonstrations.wolfram.com/33DeterminantsUsingDiagonals/

As you can see above, the determinant is calculated using the following formula for a 3x3 matrix.

Det(mat) = ( r11  *  r22  * r33 ) +  ( r12 * r23 * r31 ) + ( r13 * r21 * r32 )

- ( ( r31 * r22 * r13 ) + ( r32 * r23 * r11 ) + ( r33 * r21 * r12 )

The highlighted numbers are all the instances of values in the first column. Since there is one as a factor in each of the 6 intermediate products the end equation will always simplify down to det(mat) = 0 + 0 + 0 - (0 + 0 + 0) or det(mat) = 0.

Section 6

Figure 11: The determinant over time when moving from an arbitrary point in task space to a singularity point.

Throughout the trajectory of the end effector, the joint values are calculated using our inverse kinematics function. Each time we calculate the joint values required, we pass them to the jacobian function which calculates the jacobian and determinant. If the determinant is less than a given threshold, the function returns false meaning that we are approaching a singularity and should stop all further motion. Our threshold was 0.5 mm. So if the robot was within 0.5 mm of a singularity we would stop all further joint movements. This is shown in figure 9 in the graph on the left-hand side. For instance, in the graph, we can see that after 5 seconds of moving along the trajectory, we approached the singularity and the determinant remained the same.

Calculating the determinant of the Jacobian helps us avoid singularities. Singularities result in undefined and unpredictable motion. If the determinant of the jacobian results in 0, then joints can move at unbounded velocities and can result in the failure of the robot. Singularities are especially to be avoided since the robot cannot get out of them. The robot will essentially be stuck.

Discussion

When it came to determining the Jacobian, we initially ran into difficulty using the geometric method. We initially tried to conduct the method using degrees, but the values we kept getting were very high values. This was confusing because we expected the determinant to be close to 0. After switching to the analytical method, we got determinants that were much closer to 0 so we stuck with that solution.

The velocity kinematics used in this lab was not as smooth as the ones we used in the last lab. At times, the motion was smooth, while at other times, there was substantial jerk. In the last lab, our inverse kinematics function was solving the equations each time the function was run. This led to very jerky behavior. This lab’s velocity kinematics was smoother than the previous labs though it still displayed some jerkiness. When we were running the arm at slower speeds, the actual velocity was much slower than expected. This is likely due to the few via points we were planning to go to being so close. At higher speeds, this lab’s velocity kinematics displayed more jerkiness than at the lower speeds.
